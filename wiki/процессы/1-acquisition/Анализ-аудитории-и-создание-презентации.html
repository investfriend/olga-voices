Процесс: Анализ аудитории и создание продающей презентации

Превращаем голос аудитории в смысловую структуру презентации, которая логически подводит к покупке.

1. Цель процесса

Создать продающую презентацию для запуска продукта через глубокий анализ ЦА и формирование смысловой структуры презентации на основе реальных слов аудитории, а не предположений маркетолога.

Когда применяется:

  • Перед каждым крупным запуском продукта (марафон, интенсив, серия вебинаров).
  • Эталонный референс — запуск февраля 2026.

Что получаем на выходе:

  • Актуальный портрет аудитории «на момент запуска» (3 сегмента) — внутри смысловой карты.
  • Смысловая карта: портреты + боли / желания / возражения.
  • Логика продажи: «тема мероприятия → продукт».
  • Структура и слайды презентации.
  • Файл материалов для спикера.
  • Метрики и обновлённая эталонная база — для следующего запуска.

3. Артефакты и хранение

На момент написания процесса единая папка запусков ещё не заведена. При первом применении создаётся /launches/<YYYY-MM-формат>/ со структурой ниже. Этот шаблон будем дорабатывать практикой.

launches/
└── 2026-02-февральский-запуск/
    ├── 01-comments/              ← исходные выгрузки TG
    ├── 02-analyses/              ← результаты прогона Gemini по каждому файлу
    ├── 03-meaning-map.md         ← смысловая карта (портреты + боли + желания + возражения)
    ├── 04-sales-logic.md         ← логика продажи: тема → продукт
    ├── 05-presentation-formula.md← формула и структура презентации
    ├── 06-slides/                ← финальные слайды
    ├── 07-speaker-notes.md       ← материалы для спикера (пишутся параллельно карте)
    ├── 08-metrics.md             ← пост-анализ метрик
    └── 09-learnings.md           ← что обновить в эталонной базе

Портрет аудитории не выносится в отдельный файл. Он входит как первый раздел смысловой карты, чтобы эксперт согласовывал портрет и боли/возражения единым пакетом.

Эталонная база (накапливается между запусками): /references/audience/

  • портреты-эталон.md
  • боли-эталон.md
  • возражения-эталон.md

Эту папку также создаём при первом применении процесса. Обновляется после каждого запуска (см. этап 13).

5. Этапы

Этап 1. Сбор комментариев аудитории

Источник 1 — Telegram-канал (за последние 3 месяца).

Способ выгрузки:

  1. Telegram Lite (десктоп) — делает маркетолог самостоятельно, либо
  2. Помощь технического специалиста.

Правило: выгружать отдельными файлами (5 файлов на запуск), а не одним большим. Объединение убивает качество последующего анализа.

Источник 2 — комментарии о клубе (аудитория ближе к покупке). Источник 3 — комментарии вебинаров (аудитория с более высоким уровнем осознанности).

Эти три источника закрывают разные уровни лестницы Ханта. Источники 2–3 на момент запуска уже накоплены — кастдев текущих учеников и расшифровки эфиров ведутся постоянно, отдельно собирать не нужно.

Артефакт: папка 01-comments/ запуска с 5+ файлами TG.

Этап 2. Анализ комментариев через нейросеть

Инструмент: Gemini (на момент запуска показывает лучшие результаты для русскоязычного анализа аудитории).

Жёсткое правило: файлы анализируются по одному, последовательно. Параллельная загрузка ломает качество — нейросеть смешивает контексты и галлюцинирует смыслы.

Порядок:

  1. Открыть диалог с Gemini.
  2. Дать установочный промпт (ниже).
  3. Загрузить первый файл → дождаться результата.
  4. Загрузить второй файл → дождаться результата.
  5. Повторить для каждого файла.

Установочный промпт (используем как есть):

Мне нужна твоя помощь в том, чтобы прописать ключевые смыслы для моей продающей презентации.
У меня есть выгрузки комментариев из моего Telegram-канала и обратная связь от участников моего инвестиционного клуба, который я и хочу продавать.
Сейчас я буду отправлять файлы по одному. По каждому файлу выдели:
- основные темы, которые волнуют аудиторию;
- реальные мысли и проблемы, которые формулируют люди (с цитатами из комментариев);
- явные и скрытые мотивации;
- повторяющиеся возражения и страхи;
- с какими трудностями человек сталкивается прямо сейчас и что ему сложно преодолеть.
Не обобщай — давай конкретные формулировки из комментариев.

Артефакт: папка 02-analyses/ запуска с N файлами анализа.

Этап 3. Дополнительные источники (опросы и анкеты)

Перед каждым запуском проверяем: проводился ли опрос аудитории за последний месяц.

  • Если опрос проводился менее месяца назад — берём его результаты.
  • Если более месяца назад или не проводился — запускаем базовый опрос в TG-канале Ольги или выкладываем расширенную анкету за бонус.

Базовая универсальная сегментация:

  • пол / возраст / род деятельности (наём / предпринимательство / фриланс).

Самые ценные срезы для нас:

  • как давно ЦА в инвестициях;
  • опыт (позитивный / негативный);
  • сумма, готовая инвестировать ежемесячно;
  • предпочтительная ниша (крипта / фонды / зарубежка / недвижка);
  • с какими трудностями сейчас сталкивается человек и что ему сложно преодолеть.

Пример боевой анкеты: https://investfriend.ru/user/control/survey/update/id/388047

Артефакт: результаты опроса прикладываются к смысловой карте.

Этап 4. Определение уровня по лестнице Ханта

Анализируем три источника отдельно, потому что это разные сегменты аудитории на разных стадиях осознанности:

  • аудитория TG-канала — потенциальные клиенты, ещё только планируют покупать;
  • аудитория вебинара — выше по осознанности;
  • аудитория клуба — уже купили, фактически с лестницы ушли (приняли решение).

Важно: TG-канал ≠ автоматически 1-я ступень. Контент мог уже прогреть людей, и они могут быть на 2-й, 3-й или выше. Задача нейросети — определить конкретную текущую ступень, а не предположить «по умолчанию».

Промпт (запускаем для каждого источника отдельно):

Я сейчас отправлю комментарии конкретного сегмента аудитории.
Контекст сегмента: [например — «люди, которые сейчас в Telegram-канале и только планируют что-то покупать; они ещё ничего у нас не покупали»].
Определи, на какой конкретной ступени лестницы Ханта они находятся (1–5).
Не делай вывод по умолчанию из того, что это TG-канал — учитывай только сами комментарии.
Если в сегменте есть подгруппы на разных ступенях — дай разбивку.

Аналогично для второго и третьего источника, меняя описание контекста («люди с вебинара…», «люди, которые уже купили клуб…»).

Зачем: разные уровни требуют разной структуры презентации. Нельзя продавать «как купить» аудитории, которая ещё не осознаёт проблему.

Этап 5. Портрет аудитории (внутри смысловой карты)

Принципы:

  1. Портрет формируется в текущий момент времени, а не «вообще». Состояние ЦА в момент конкретного запуска.
  2. Не абстрактный портрет — конкретные 3 сегмента (по самым частым пересечениям ответов).

Что фиксируем для каждого сегмента:

  • проблемы, которые волнуют;
  • желания и цели;
  • страхи / тревоги;
  • триггеры покупки.

Портрет не выводится в отдельный артефакт и не согласовывается отдельно. Он становится первым разделом смысловой карты (этап 6) и согласовывается уже в её составе.

Этап 6. Смысловая карта

Главный рабочий документ всего запуска. Что входит:

  1. 3 портрета — финальные, готовые для работы (см. этап 5).
  2. Самые популярные боли — формулировки из реальных комментариев, не выдуманные.
  3. Самые популярные запросы (желания / цели).
  4. Самые популярные возражения — что мешает покупке.

Источник возражений: в первую очередь — комментарии, во вторую — анализ конкурентов (см. сотрудник).

С картой далее работают маркетолог и копирайтер — на её основе пишутся презентация, письма, дожимы, рассылки. Финальные формулировки оффера — за маркетологом.

Контрольная точка: карта (включая портрет) согласовывается с экспертом по продукту или руководителем перед переходом к логике продажи.

Артефакт: 03-meaning-map.md.

Каждое возражение из карты позже привязывается к конкретному слайду или тейку в презентации (см. этап 8).

Этап 7. Формирование продающего мероприятия

На основе смысловой карты выбираем формат мероприятия:

  • марафон;
  • интенсив;
  • серия вебинаров;
  • «другая активность» (новый формат).

В запуске февраля 2026 использовались два вебинара.

Формат должен соответствовать готовности аудитории по Ханту (этап 4).

Этап 8. Логика продажи (тема → продукт)

Анализируем аудитория ↔ продукт (например, ИнвестКлуб). Цель — определить, через какие смыслы лучше всего продавать продукт именно этой аудитории прямо сейчас.

Промпт-шаблон:

На основе портрета аудитории и описания продукта построй логику продажи:
какие смыслы и в какой последовательности лучше всего использовать,
чтобы аудитория с темы вебинара логически перешла к покупке продукта.

Контрольная точка: логику продажи согласовывает продюсер.

Артефакт: 04-sales-logic.md.

Делается в тот же день, что и смысловая карта.

Этап 9. Структура презентации

На основе логики продажи формируется формула презентации — связка:

  • контента;
  • пути аудитории;
  • решения проблемы.

Параллельно для каждого возражения из смысловой карты определяется конкретный слайд / тейк, который его закрывает. Без этой привязки карта остаётся на этапе анализа.

Артефакт: 05-presentation-formula.md.

Этап 10. Работа со слайдами

Принцип: на слайде — минимум информации. Основная мысль раскрывается в речи спикера.

  • На слайде — ключевая мысль (1 заголовок + 1 тезис).
  • Подробности — у спикера в 07-speaker-notes.md.

Если слайд можно убрать без потери смысла — его убираем.

Визуализация смыслов. Принцип: «всё, что можно показать, — нужно показывать». Используем картинки, примеры, иллюстрации, мемы, графики, диаграммы.

Образы аудитории — ключевая часть. Подбор по:

  • увлечениям ЦА;
  • образу жизни;
  • внешнему портрету.

Аудитория должна узнавать себя на изображениях.

Эмоциональные примеры: используем сильные сценарии (родители не могут оплатить обучение детям, страх потерять накопления и т. п.) — они усиливают вовлечённость.

Этап 11. Материалы для спикера

Вся дополнительная информация, которая не отражена на слайдах, но раскрывает тему и погружает в контекст, выгружается в 07-speaker-notes.md.

В файле — на каждый слайд:

  • ключевой тезис слайда;
  • 2–4 факта / истории / аргумента для речи;
  • ответы на типовые возражения именно по этому слайду.

Материалы для спикера пишутся параллельно смысловой карте, по ходу анализа — а не отдельным этапом в конце.

Спикер использует этот файл как опору при подготовке.

Этап 12. Проверка комплаенса и единого смыслового поля

Перед эфиром презентация проходит обязательный чек:

Комплаенс (специфика инвестиционной тематики):

  • нет прямых инвестиционных рекомендаций;
  • нет конкретных акций с оценкой «купить / продать»;
  • нет прогнозов движения рынка;
  • нет фраз, которые регулятор может квалифицировать как индивидуальную инвестрекомендацию.

Единое смысловое поле — проверка консистентности всех касаний с аудиторией:

  • презентация;
  • вебинары;
  • рассылки;
  • маркетинговые материалы;
  • посты в TG-канале;
  • лендинг.

Все должны звучать как одно сообщение, усиливая друг друга. Каждый новый материал дополняет предыдущий, а не противоречит.

Этап 13. Метрики и обновление эталонной базы

После запуска проводим детальный разбор результатов.

Что анализируем:

  1. Конверсии на каждом этапе (заявка → регистрация → дошёл до эфира → дошёл до оффера → купил).
  2. Выполнение / невыполнение плановых показателей. Для каждого «не дотянули» / «перевыполнили» — гипотеза почему.
  3. Какие смыслы / слайды зашли лучше всего (по обратной связи и поведению на эфире).
  4. Какие возражения остались не закрыты (по чату, личке, отказам после оффера).

Пример таблицы метрик: https://docs.google.com/spreadsheets/d/1arX9SMs6r06WtqqykJ5HtI1UYwJExVWhp4Lo_N-P4yM/edit?usp=sharing

Главное правило этапа — обновляем эталонную базу:

  • новые подтверждённые портреты → /references/audience/портреты-эталон.md;
  • новые боли → /references/audience/боли-эталон.md;
  • новые возражения и формулировки их закрытий → /references/audience/возражения-эталон.md;
  • лучшие смыслы и слайды, которые сработали, — фиксируем как референс для следующего запуска.

Без этого этапа процесс каждый раз стартует «с нуля» и теряет компаунд-эффект.

Артефакты: 08-metrics.md, 09-learnings.md + обновлённые файлы в /references/audience/.

Этап 1. Сбор данных

  • определена дата запуска
  • определён период анализа (3 мес)
  • выгружены комментарии TG-канала (5 файлов)
  • подтянуты комментарии о клубе и вебинаров (из текущего кастдева)
  • проверено: опрос аудитории не старше месяца. Если старше — запущен новый

Этап 2. Анализ

  • прогнаны все файлы по очереди через Gemini установочным промптом
  • определён уровень по лестнице Ханта отдельно для 3 источников (с явным указанием контекста сегмента в промпте)

Этап 3. Смысловая карта

  • портрет (3 сегмента) прописан как первый раздел смысловой карты
  • собраны боли / желания / возражения с цитатами
  • добавлены инсайты из анализа конкурентов
  • смысловая карта согласована с экспертом по продукту или руководителем

Этап 4. Продукт и логика продажи

  • выбран формат мероприятия

  • на слайдах оставлено минимум информации
  • подготовлены материалы для спикера (писались параллельно смысловой карте)

Этап 6. Контроль перед запуском

  • юридический / комплаенс-чек пройден
  • проверено единое смысловое поле всех касаний
  • финальные формулировки оффера утверждены маркетологом

Этап 7. После запуска

  • собраны метрики, заполнен 08-metrics.md
  • разобраны причины отклонений от плана
  • эталонная база в /references/audience/ обновлена
  • зафиксированы лучшие смыслы и слайды
