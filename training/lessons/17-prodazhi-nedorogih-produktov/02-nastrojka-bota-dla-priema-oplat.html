Клуб по подписке: Prodamus + BotHelp + Telegram

Резюме: Инструкция описывает полную настройку автоматического клуба по подписке — от активации функционала в Prodamus до сборки трёхветочного бота в BotHelp, который управляет доступом участников в закрытый Telegram-чат.

Что понадобится

  • Аккаунт в Prodamus с активированным модулем подписок
  • Аккаунт в BotHelp (платная версия)
  • Закрытый Telegram-чат или канал для участников клуба
  • Отдельный Telegram-бот для приёма оплат (создаётся через BotFather)
  • Google Таблица для хранения данных оплативших

Шаг 1. Активация модуля подписок в Prodamus

  1. Если Prodamus ещё не подключён — подключите его через менеджера. При подключении сразу сообщите, что нужен клуб по подписке (активация стоит 1 000 ₽).
  2. Если Prodamus уже есть — напишите в поддержку с просьбой активировать клуб по подписке. Стоимость активации — 2 000 ₽.
    • Поддержку найдёте внизу личного кабинета Prodamus (иконка чата → WhatsApp / Telegram).
  3. При активации отправьте менеджеру правила клуба. Образец структуры:
    • Для чего создан клуб
    • Что получает участник при оплате (доступ в закрытую группу Telegram, базу знаний, еженедельные групповые Zoom-разборы и т.д.)
    • За что могут исключить (например: публикация материалов в открытый доступ)
    • Как покинуть клуб (написать в Telegram)
  4. После оплаты и согласования правил в вашем кабинете Prodamus появится раздел «Подписки» в настройках.

Шаг 2. Создание подписки в Prodamus

  1. Откройте раздел Настройки → Подписки → нажмите «Добавить подписку».
  2. Заполните поля:
    • Название подписки: например, «Доступ к мастер-майнду на 30 дней»
    • Описание: например, «Обучение продажам в Telegram в формате прямых эфиров в закрытом Telegram-чате. Доступ на 30 дней»
    • Стоимость: укажите цену (например, 990 ₽)
    • Стоимость первого платежа: можно сделать выше (например, 5 000 ₽ — первый, 1 500 ₽ — последующие), чтобы снизить отток. Учтите: при высоком первом платеже вернувшиеся участники будут платить его снова.
    • Количество автосписаний: оставьте пустым (бессрочно) — пока участник сам не отпишется, деньги будут списываться ежемесячно.
  3. Сохраните подписку. Скопируйте её ID — он понадобится в BotHelp.

Настройки попыток списания

В разделе Настройки подписки укажите:

  • Интервал попыток списания — например, каждые 8 часов
  • Количество попыток — например, 10 Если за 10 попыток деньги не списались — участник автоматически считается отписанным.

Шаг 3. Создание платёжной страницы в BotHelp

  1. Перейдите: Автоматизация → Приём платежей → Новая платёжная страница → Prodamus.
  2. Выберите тип: Подписка.
  3. Заполните поля:
    • ID подписки — скопируйте из Prodamus (Шаг 2)
    • Название домена — часть URL вашего кабинета Prodamus до .payform.ru. Пример: если адрес paypage.payform.ru, домен — paypage
    • Секретный ключ — найдёте в Настройках Prodamus (скопируйте и вставьте)
  4. Заполните описание заказа. Обязательно используйте слово «обучение» (особенно если работаете на патенте): > «Обучение продажам в Telegram в формате прямых эфиров в закрытой группе в Telegram. Доступ на 30 дней»
  5. Укажите:
    • Страница при неудачной оплате — ссылка на страницу с предложением написать в Telegram для помощи
    • Страница при успешной оплате — ссылка на ваш Telegram-бот (формат: t.me/имя_бота)
  6. Сохраните. Скопируйте идентификатор платёжной страницы (например, payment_1) — понадобится в боте.

Шаг 4. Получение ID закрытого Telegram-чата

  1. Зайдите в закрытый чат → Изменить → Администраторы → выберите себя → включите «Анонимность».
  2. Напишите любое тестовое сообщение от имени чата.
  3. Перейдите в Telegram, найдите бота @getmyid_bot.
  4. Перешлите это тестовое сообщение боту @getmyid_bot.
  5. Бот вернёт строку вида: Forwarded from chat: -100XXXXXXXXXX — это и есть ID чата (со знаком минус).
  6. Скопируйте ID. Отключите анонимность в настройках администратора. Удалите тестовое сообщение из чата.

Шаг 5. Копирование и настройка бота в BotHelp

> Ссылка на шаблон бота — в описании к этому уроку.

  1. Перейдите по ссылке на шаблон бота → нажмите «Копировать к себе в BotHelp».
  2. Бот вставится в ваш аккаунт. Откройте его — потребуется настроить метки и ID чата.

5.1. Создание меток

Создайте две метки (Labels):

  • заблокирован в чате — присваивается, когда участник удалён из чата за неоплату
  • оплатил клуб — присваивается при успешной оплате Как создать метку: в BotHelp найдите нужный элемент → «Добавить метку» → введите название.

5.2. Настройка элемента «Оплатил» (главный блок)

  1. Найдите элемент «Оплатил» в боте.
  2. В блоке «Условие: метка» выберите метку заблокирован в чате.
  3. Сохраните.

5.3. Вставка ID чата

Найдите все блоки, где упоминается закрытый чат (разблокировка, проверка участия, подтверждение заявки). В каждом из них вставьте ID чата из Шага 4.

5.4. Вставка ссылки-приглашения в чат

  1. Откройте закрытый чат → Тип группы → Частная → Скопировать ссылку.
  2. Вставьте эту ссылку во все блоки бота, где отправляется приглашение в чат.

Шаг 6. Логика трёх веток бота

Бот работает по трём независимым веткам:

Ветка 1 — Первый запуск бота (новый пользователь)

  1. Участник активирует бота → приходит приветственное сообщение с описанием клуба и стоимостью.
  2. Через 3 секунды — проверка: есть ли email в базе?
    • Есть → отправляется ссылка на оплату (payment_1)
    • Нет → задаётся вопрос: «Напишите email — на него придёт чек об оплате» → ответ сохраняется в поле email
  3. После получения email → ссылка на оплату.
  4. Если участник не ответил в течение 140 минут → напоминание.
  5. Если не ответил в течение 24 часов → бот останавливается для этого пользователя. Серия напоминаний (если не оплатил):
  6. +30 минут: проверка метки оплатил клуб
    • Есть метка → всё ок
    • Нет метки → сообщение: «Вижу, что вы ещё не оплатили доступ. Напоминаю: [перечень преимуществ]. Оплатить можно по ссылке: [ссылка]»
  7. Можно добавить ещё одно напоминание через нужный интервал

Ветка 2 — Участник оплатил

  1. Добавить метку оплатил клуб.
  2. Проверить: есть ли метка заблокирован в чате?
    • Да → Разблокировать в чате (вставить ID чата) → отправить: «Благодарю за продление! Подайте заявку на вступление в чат: [ссылка]» → подождать 1 минуту → подтвердить заявку → проверить, вступил ли → повторять до вступления
    • Нет → Проверить: состоит ли в чате?
      • Да → отправить: «Благодарю за продление подписки!»
      • Нет → отправить: «Поздравляю с оплатой! Вступите в чат по ссылке: [ссылка]» → подтвердить заявку
  3. Экспорт в Google Таблицу (имя, Telegram, email — см. Шаг 7).
  4. Удалить метку заблокирован в чате.
  5. Через 27 дней → напоминание: «Через 3 дня автоматически спишется оплата за продление. Если хотите остановить подписку — напишите мне: [ссылка на Telegram]»

Ветка 3 — Участник отписался / не оплатил

  1. Бот удаляет участника из закрытого чата.
  2. Добавить метку заблокирован в чате.
  3. Удалить метку оплатил клуб.
  4. Отправить сообщение: «Вижу, что подписка остановилась. Если захотите вернуться — оплатить можно по ссылке: [ссылка на оплату]» При ошибке списания: запустить Ветку 3 (отписался) + дополнительно отправить: «К сожалению, не получается списать деньги. [инструкция]»

Шаг 7. Настройка экспорта в Google Таблицу

  1. Создайте Google Таблицу с колонками: Ник в Telegram, Имя, Email (можно добавить Источник и другие нужные поля).
  2. В BotHelp, в блоке экспорта, нажмите на карандаш → вставьте ссылку на таблицу → дайте доступ BotHelp к Google Таблицам.
  3. Сопоставьте поля:
    • username → Ник в Telegram
    • name / full_name → Имя
    • email → Email

Шаг 8. Подключение вебхука в Prodamus

  1. В Prodamus: Настройки подписки → URL-адрес для уведомлений о свершении оплат.
  2. Вставьте туда адрес BotHelp в формате, указанном в описании к этому уроку.
  3. Сохраните. Теперь при каждой оплате Prodamus будет отправлять уведомление в BotHelp и автоматически запускать нужную ветку бота.

Шаг 9. Привязка платёжной страницы к боту

  1. В BotHelp: Автоматизация → Приём платежей → откройте созданную платёжную страницу.
  2. Добавьте:
    • Действие при подписке → Запустить бота → выбрать шаг «Оплатил»
    • Действие при завершении подписки → Запустить бота → выбрать шаг «Отписался»
    • Действие при ошибке → Запустить бота → выбрать шаг «Отписался»

Шаг 10. Создание отдельного Telegram-бота для оплат

  1. Создайте нового бота через BotFather (инструкция — в модуле по литмагнитной воронке, раздел «Сборка Telegram-бота»).
  2. Назовите бота понятно: например, «Оплата клуба [название]».
  3. В BotHelp: Настройки → Каналы → Добавить новый канал → Telegram → вставьте API-токен от BotFather.
  4. Привяжите созданный бот к собранной воронке.

Как давать ссылку участникам

  • В Telegram (например, в вашем канале) — давайте Deep Link из раздела Настройки → Расширенные настройки бота в BotHelp. Он сразу запускает нужный шаг без кнопки «Старт».
  • Вне Telegram (Instagram, ВКонтакте и т.д.) — давайте обычную ссылку t.me/имя_бота. Участник нажмёт «Старт» и попадёт в ту же воронку.

Управление подписками вручную

  • Чтобы отписать участника по его просьбе: Prodamus → Подписки → Подписчики → найдите нужного → переключите ползунок в неактивное состояние.
  • Чтобы закрыть клуб для всех: Prodamus → Подписки → отключите все активные подписки. Участники доживают текущий оплаченный период, новых списаний не будет.

Интеграция BotHelp + ProDamus: бот приёма оплаты

Резюме: Инструкция описывает полную настройку бота в BotHelp, который отправляет ссылку на оплату, открывает доступ к продукту после оплаты и автоматически дожимает тех, кто не заплатил.

Необходимые инструменты

  • BotHelp — платная версия (не пробная, не бесплатная)
  • ProDamus — для приёма платежей
  • BotFather — для создания Telegram-бота (нужен только при запуске в блоге)
  • Google Таблицы — для хранения контактов оплативших

Шаг 1. Подключение ProDamus к BotHelp

  1. Войдите в BotHelp → Настройки (левое меню) → Интеграции.
  2. Найдите раздел ProDamus.
  3. Скопируйте ваш домен из личного кабинета ProDamus (отображается вверху — например, paypage.payform.ru). Вставьте в поле домена.
  4. В ProDamus перейдите в Настройки → найдите раздел Секретный ключ → скопируйте его.
  5. Вставьте секретный ключ в BotHelp. Нажмите Сохранить.

Шаг 2. Создание платёжной страницы

BotHelp → Автоматизация → Приём платежей → Создать новую платёжную страницу

Заполните поля:

| Поле | Значение | |---|---| | Система приёма платежей | ProDamus | | Тип приёма платежей | Продукт (разовый платёж) | | Цена | Например, 990 рублей | | Название товара | Обязательно содержит слово «обучение» — например: «Обучение продажам в Telegram» | | Ваш домен | Домен ProDamus (тот же, что в шаге 1) | | Секретный ключ | Секретный ключ ProDamus |

Передача данных:

  • Номер телефона из профиля — включить
  • Email из профиля — включить
  • UTM-метки из профиля — включить Описание заказа: Прописать детально — например: «Обучение продажам в Telegram в формате трёх эфиров в закрытом Telegram-канале». Это важно для чеков и при возможных спорах. Срок действия ссылки:
  • Без ограничений — если продукт продаётся постоянно
  • Конкретная дата/время — если это запуск с дедлайном (например, до 23:59 24 ноября) Страница после оплаты: Вставьте ссылку на ваш Telegram-бот в формате t.me/имя_бота. Это перебросит пользователя обратно в бот сразу после оплаты. Страница при неудачной оплате: Укажите соответствующую страницу из ProDamus. Нажмите Сохранить. Переименуйте платёжную страницу — например: Платёжка интенсив 990.

Шаг 3. Создание бота в BotFather (только для запуска в блоге)

> Если продаёте мини-продукт через воронку (не через канал/блог) — этот шаг пропустите.

  1. Создайте нового бота в BotFather (подробная инструкция — в модуле по созданию лид-магнитной воронки).
  2. В BotHelp перейдите: Настройки → Каналы → Добавить новый канал → Telegram.
  3. Скопируйте API-токен из BotFather, вставьте в BotHelp. Сохраните. Зачем: Отдельный бот нужен, чтобы ссылка из канала не смешивалась с лид-магнитным ботом. Иначе пользователи получат лид-магнит вместо ссылки на оплату.

Шаг 4. Копирование шаблонного бота

  1. Перейдите по ссылке, прикреплённой к этому уроку (шаблон бота для оплаты).
  2. Нажмите «Копировать в BotHelp» (правый нижний угол).
  3. Бот загрузится в ваш кабинет BotHelp. Переименуйте его — например: Оплата интенсива.

Шаг 5. Привязка бота к Telegram-каналу

BotHelp → Автоматизация → Приветствие → Новое приветствие

  1. Выберите бот (тот, что добавили в шаге 3).
  2. В поле «Рассылка» выберите Оплата интенсива (скопированный бот). Если рассылка недоступна для выбора:
  3. Убедитесь, что рассылка активна (три точки рядом → Активировать).
  4. Убедитесь, что рассылка привязана к нужному Telegram-боту: Настройки → Канал → поменять бот.

Шаг 6. Логика бота изнутри

Бот работает по следующей схеме:

Ветка А — email уже есть в базе BotHelp

``` Пользователь нажимает /start → Бот проверяет поле email → Email найден → отправляет ссылку на оплату ```

Сообщение: «Отлично, вот ссылка на оплату. Перейдите по ней, оплатите 990 рублей — я сразу добавлю вас в закрытый канал.»

Как вставить ссылку на оплату в сообщение:

  • BotHelp → Автоматизация → Приём платежей → найдите созданную платёжную страницу
  • Скопируйте ссылку вида payment_1 или payment_2
  • Вставьте прямо в текст сообщения — BotHelp подставит нужное значение автоматически

Ветка Б — email в базе нет

``` Email не найден → отправляет вопрос через элемент «Вопрос» (не «Текст») ```

Настройка элемента Вопрос:

  • Текст: «Привет! Чтобы оплатить участие, напишите в ответном сообщении ваш email — на него придёт чек об оплате.»
  • Сохранить ответ в поле: email Настройки ожидания ответа (раздел Настройки внутри шага):
  • Ожидать ответа: 24 часа
  • Напоминание через: 126 минут — текст: «Жду вашу почту. Как только пришлёте email, скину ссылку на оплату интенсива, а на почту придёт чек об оплате.» Если пользователь не ответил за 24 часа:
  • Действие: Остановить бот Оплата интенсива Если ответил (email записан):
  • Отправить сообщение со ссылкой на оплату (аналогично Ветке А)

Шаг 7. Настройка меток и дожимающей цепочки

Создание метки

В шаге бота добавьте действие «Присвоить метку» — например: Оплатил интенсив ноябрь.

Дожимающая цепочка (для тех, кто не оплатил)

Через 30 минут после отправки ссылки:

  • Проверить: есть ли метка Оплатил интенсив ноябрь
  • Если есть — ничего не отправлять
  • Если нет — отправить: «Вижу, что вы ещё не оплатили интенсив. Сделать это можно по ссылке: [ссылка на оплату]. [Описание ценности продукта]» На следующий день в 9:00:
  • Снова проверить метку
  • Если нет — отправить: «Последняя возможность присоединиться к интенсиву. [ссылка]»

После оплаты — присвоение метки

  • BotHelp → Автоматизация → Приём платежей → платёжная страница → Действия → Добавить действие
  • Выбрать: Запустить бот Оплата интенсива, шаг: Присвоить метку об оплате
  • Сохранить Что происходит при оплате:
  • Пользователю присваивается метка Оплатил интенсив ноябрь
  • Уходит сообщение: «Благодарю за оплату! Вступайте в закрытый канал по ссылке: [ссылка на канал]»
  • Данные экспортируются в Google Таблицу
  • Бот Оплата интенсива для этого пользователя — останавливается

Шаг 8. Экспорт оплативших в Google Таблицу

  1. Создайте Google Таблицу со столбцами: Ник в Telegram, Имя, Почта.
  2. В BotHelp вставьте ссылку на таблицу.
  3. Разрешите BotHelp доступ к Google Таблицам при запросе.
  4. В настройках экспорта сопоставьте поля:
    • Ник в TG → username
    • Имя → полное имя
    • Почта → email

Шаг 9. Получение ссылки на бот для размещения

BotHelp → бот Оплата интенсива → Настройки → Расширенные настройки → Ссылка на бот

Используйте эту ссылку:

  • В воронке: вставьте в нужный шаг цепочки как кнопку или текстовую ссылку
  • В блоге/канале: вставляйте в посты при каждом запуске Ссылка ведёт сразу в нужный шаг бота.

Повторный запуск (следующий месяц / следующий интенсив)

Если меняете цену:

BotHelp → Автоматизация → Приём платежей → нужная платёжная страница → изменить цену. Ссылка остаётся той же, цена обновится автоматически для всех, включая тех, кто уже получил сообщение, но не оплатил.

Если запускаете новый поток:

  1. В шаге «Присвоить метку» — удалить старую метку, создать новую: Оплатил интенсив декабрь
  2. В шагах проверки метки — выбрать новую метку (Оплатил интенсив декабрь)
  3. Обновить ссылку на закрытый канал (если для нового потока создаётся новый канал)

Если продаёте мини-курс в записи (без дат):

Метку не менять. Просто давайте ссылку на бот в постах — система работает без изменений.

Важные замечания

  • Название товара в ProDamus и BotHelp всегда должно содержать слово «обучение» — это требование для налоговой при патентной системе налогообложения.
  • Описание заказа должно точно отражать, что продаётся — формат, канал, количество эфиров/уроков.
  • Цена 990 рублей показывает наибольшую конверсию для мини-продуктов по сравнению с 1490, 2990 и 4990 рублями.
