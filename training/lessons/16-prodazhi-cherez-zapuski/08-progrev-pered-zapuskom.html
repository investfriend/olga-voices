Видеоконспект

Прогрев аудитории перед запуском

Запуск работает только при наличии подготовленной аудитории. Этот урок объясняет, когда прогрев нужен, какие смыслы транслировать и как использовать контент-план для системного ведения блога.

Когда можно запускаться сразу

Запуск проводите без дополнительного прогрева, если выполняются все условия:

  • Блог ведётся стабильно минимум 3–4 недели
  • Аудитория уже интересуется продуктом — задаёт вопросы, пишет в личку
  • Вы периодически упоминаете продукт (например: «продажи пока закрыты, если интересно — пишите») Если хотя бы одно условие не выполнено — сначала прогрев.

Когда нужен прогрев перед запуском

Прогрев необходим, если:

  • Блог только начат
  • Блог вёлся под другую тему или другой продукт
  • Блог вёлся в «лайв-формате» без упоминания продукта Длительность: 2–4 недели.

Ключевые смыслы для прогрева

Через контент необходимо донести следующие идеи:

  1. Изменения, которые даёт продукт — люди покупают не набор видео, а трансформацию. Показывайте, как меняется жизнь клиентов.
  2. Суть вашего подхода — раскройте, за счёт чего клиент получает результат и в чём уникальность методологии.
  3. Работа над продуктом — показывайте процесс: доработки, улучшения, созвоны с клиентами.
  4. Ваша экспертность — демонстрируйте, что вы реально разбираетесь в теме, а не просто обучаете.
  5. Результаты клиентов — кейсы, отзывы, конкретные достижения.

Контент-план: структура недели

Используйте следующий рубрикатор как ориентир (не жёсткое правило):

| День | Формат | |---|---| | Понедельник | Скрин + текст / фото + текст / текстовый пост | | Вторник | Подкаст (фото + подкаст или только подкаст) | | Среда | Текстовый пост | | Четверг | Прямой эфир | | Пятница | Текстовый пост | | Воскресенье | Видео |

По видео: публикуйте в Telegram, ВКонтакте и YouTube одновременно — работает на органику на всех платформах.

Как использовать таблицу контент-плана в Notion

  1. Зарегистрируйтесь в Notion (если нет аккаунта).
  2. Перейдите по ссылке под видео и нажмите «Копировать» — таблица появится в вашем Notion.
  3. Снимите все галочки с идей.
  4. Выберите одну идею на день — ту, которая сейчас больше откликается.
  5. Напишите и опубликуйте пост.
  6. Поставьте галочку напротив использованной идеи.
  7. При следующей публикации выбирайте идеи без галочек — чтобы не повторяться. Как выбирать идею из таблицы:
  8. Пробегитесь по списку смыслов.
  9. Выберите тот, о котором хотите рассказать сегодня (кейс, команда, процесс работы, вложения в образование и т.д.).
  10. В таблице для каждой идеи есть: ключевая мысль + способы усиления + пример реального поста. Минимум для прогрева: 10–15 постов из таблицы, не обязательно все.

Принцип работы блога

Каждый пост должен продавать продукт — напрямую или косвенно. Примеры подходящих тем:

  • Участие в мастермайнде или обучении → подсвечивает экспертность
  • Встреча с клиентами → показывает уровень продукта
  • Личный пост о заряженности и результатах → продаёт вас как эксперта
  • История, как продали продукт другу → демонстрирует уверенность в качестве
  • Отзывы клиентов → социальное доказательство
  • Анонс повышения цены → стимулирует принятие решения
  • Кейс клиента (например, купил машину отцу) → результат работы с вами Правило: даже если в посте нет прямого призыва к покупке — он должен подсвечивать вашу экспертность, серьёзность подхода или результаты клиентов.

Следующий шаг

Скопируйте таблицу в Notion и начните публиковать посты сегодня — по одной идее в день, каждый раз продавая идею работы с вами.
